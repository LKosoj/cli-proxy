# dotenv_loader

Модуль предоставляет инструменты для парсинга и загрузки переменных окружения из `.env`-файлов. Поддерживает базовый синтаксис: ключ=значение, экранирование кавычками, inline-комментарии (только при наличии пробела перед #), директиву `export`. Функции позволяют загружать переменные в `os.environ` с опцией контроля перезаписи существующих значений.

Ключевые структуры данных  
Dict[str, str] — словарь переменных окружения, где ключ — имя переменной, значение — её содержимое

parse_dotenv(text: str) -> Dict[str, str]  
Парсит текст .env-файла и возвращает словарь переменных.  
Аргументы  
text — строка с содержимым .env-файла, построчно  
Возвращает  
Словарь пар ключ-значение, извлечённых из текста

---  
load_dotenv(path: str, override: bool = False) -> Dict[str, str]  
Загружает переменные из .env-файла в os.environ и возвращает применённые значения.  
Аргументы  
path — путь к .env-файлу  
override — если True, перезаписывает существующие переменные в окружении  
Возвращает  
Словарь переменных, которые были установлены (или должны быть установлены при override=False)  
Исключения  
Игнорирует ошибки чтения файла и возвращает пустой словарь при проблемах

---  
load_dotenv_near(path: str, filename: str = ".env", override: bool = False) -> Dict[str, str]  
Загружает .env-файл из директории, указанной в пути к файлу (например, из директории скрипта).  
Аргументы  
path — базовый путь, на основе которого определяется директория  
filename — имя .env-файла (по умолчанию ".env")  
override — флаг перезаписи существующих переменных окружения  
Возвращает  
Словарь переменных, загруженных в окружение (или которые были бы загружены)
