agent/approvals/blocked-patterns.json

Конфигурационный файл предназначен для определения правил блокировки потенциально опасных команд и выражений в системе с целью предотвращения утечек данных, несанкционированного доступа, выполнения вредоносного кода и других угроз безопасности. Правила основаны на регулярных выражениях и сопровождаются метаданными, такими как категория угрозы и причина блокировки, что обеспечивает гибкое управление политиками безопасности и прозрачность срабатываний.

Секции и ключи

id — уникальный идентификатор правила, используется для ссылок и логирования
category — категория угрозы, группирующая правила по типу риска (например, exfiltration, code_execution, privilege, obfuscation, crypto_mining)
pattern — регулярное выражение, определяющее запрещённый шаблон команды или кода
flags — флаги регулярного выражения, например "i" для регистронезависимости
reason — текстовое пояснение причины блокировки, отображаемое при срабатывании правила

Важные параметры

pattern — ключевой элемент правила, определяющий точность и чувствительность обнаружения, должен быть тщательно составлен для минимизации ложных срабатываний
category — позволяет классифицировать инциденты и применять соответствующие меры реагирования, а также упрощает анализ логов
reason — информирует пользователя и администратора о причине блокировки, должен быть понятным и содержательным
